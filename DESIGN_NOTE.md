# Design Decision Note — Widget-to-Chat Dashboard

To ensure users always know which widget is active, I used redundant visual signals including a prominent blue border, a flash animation, and a spring-animated "Selected" badge upon clicking. Context transfer is handled via a persistent banner at the top of the chat panel that mirrors the selected widget's specific icon, color, and key metrics, creating a visible handshake between the data and the AI. To solve the "cold start" problem and guide users, each widget immediately displays an example question and a structured AI response card containing an inline chart when first clicked. Finally, switching between widgets triggers a crossfade and a subtle blue overlay pulse in the chat panel to clearly communicate that the context has changed.
